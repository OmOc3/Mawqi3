الملخص التنفيذي

صورة مختصرة لما يقدمه المشروع ولماذا هو مهم للعميل.

إيكوبست هو نظام ويب عربي مخصص لشركات مكافحة الآفات التي تدير محطات الطعوم وتحتاج إلى ربط العمل الميداني بالإدارة والمتابعة. المنصة تحول زيارة المحطة من إجراء ورقي متفرق إلى دورة عمل رقمية واضحة: تعريف المحطة، الوصول لها عبر QR، تسجيل حالة الفحص، مراجعة التقرير، متابعة الاستثناءات، ثم تصدير البيانات وتحليلها.

القيمة الأساسية: النظام يقلل فقدان البيانات، يرفع التزام الفريق بإجراءات الفحص، ويمنح الإدارة رؤية فورية عن المحطات النشطة، التقارير المنتظرة، أداء الفنيين، وطلبات العملاء.

دورة العمل التشغيلية

الطريقة التي يستخدم بها الفريق النظام يوميًا.

إنشاء المحطة وتحديد بياناتها المدير ينشئ محطة برقم واضح، اسم، موقع، منطقة، وصف، وإحداثيات اختيارية.
توليد QR لكل محطة كل محطة لها رابط QR مستقل يفتح مسار التقرير الخاص بها ويقلل إدخال البيانات يدويًا.
الفني يسجل نتيجة الزيارة الفني يحدد نوع الآفة، حالة المحطة والطعم، الملاحظات، وبيانات الصور عند تفعيل التخزين.
تحديث بيانات المحطة تلقائيًا بعد إرسال التقرير يتم تحديث آخر زيارة، عدد التقارير، والفني المسؤول ضمن معاملة قاعدة بيانات واحدة.
المشرف أو المدير يراجع التقرير التقارير تنتقل بين بانتظار المراجعة، تمت المراجعة، أو مرفوض مع ملاحظات واضحة.
متابعة وتشغيل وتحليل لوحات التحكم تعرض الاستثناءات، الفلاتر، الحضور، الشيفتات، الرواتب، وملفات التصدير.
نقطة قوة مهمة: إرسال التقرير ليس مجرد حفظ صف في قاعدة البيانات؛ العملية تشمل إنشاء التقرير، حفظ الحالات، تحديث المحطة، وكتابة سجل تدقيق داخل Transaction واحدة لضمان اتساق البيانات.
المرحلة المسؤول مخرجات المرحلة
تجهيز المحطات المدير محطات نشطة، QR جاهز، بيانات موقع ومنطقة.
الفحص الميداني الفني تقرير زيارة مرتبط بالمحطة والفني والوقت.
المراجعة المشرف أو المدير اعتماد، رفض، أو ملاحظات متابعة.
الإدارة المدير تحليلات، تصدير، إدارة فريق، ومراجعة سجل التدقيق.
دورة العمل مبنية على المسارات والـ Server Actions الموجودة في المشروع الحالي.
4

الوظائف الرئيسية

نطاق المنتج من منظور العميل والمستخدمين.

إدارة محطات الطعوم

إنشاء وتعديل وتعطيل المحطات، توليد أرقام تلقائية، تحديد المنطقة والموقع، إضافة إحداثيات، وتصدير QR للطباعة والاستخدام الميداني.

تسجيل تقارير الفحص

نموذج عربي واضح للفنيين يشمل حالة المحطة، حالة الطعم، أنواع الآفات، الملاحظات، والربط المباشر بمحطة محددة.

مراجعة واعتماد التقارير

المشرفون والمديرون يراجعون التقارير، يضيفون ملاحظات، ويحولون حالة التقرير إلى معتمد أو مرفوض أو قيد المراجعة.

لوحات تحكم للإدارة والإشراف

صفحات منفصلة للمدير والمشرف تعرض التقارير، المحطات، النشاط، الطلبات، الحضور، الشيفتات، المهام، والتحليلات.

بوابة العملاء وطلبات الخدمة

إنشاء حسابات عملاء، تسجيل طلبات محطة أو متابعة، مراجعة الطلبات من الإدارة، وربط العميل بالمحطات المصرح له بمشاهدتها.

الحضور والانصراف والشيفتات

تسجيل بداية ونهاية الشيفت، التحقق من القرب من المحطات، متابعة الحضور، ربط الشيفت بالمدة المتوقعة، ودعم مراجعة الرواتب.

التقارير اليومية

توثيق ملخص يومي للفني، المحطات التي تمت تغطيتها، الصور عند اعتماد التخزين، والملاحظات المرتبطة بالعمل اليومي.

التصدير والتحليلات

تصدير CSV للتقارير والحضور، مؤشرات تشغيلية، وملخصات تساعد الإدارة على معرفة المحطات المتأخرة أو التقارير التي تحتاج مراجعة.

تجربة استخدام عربية: الواجهة مبنية باتجاه RTL، نصوص عربية، أسلوب تشغيلي مختصر، وألوان حالات واضحة مثل بانتظار المراجعة، تمت المراجعة، مرفوض، نشط، وغير نشط.
نطاق الوظائف يعرض منصة الويب ولوحات التشغيل الحالية للعميل.

الأدوار والصلاحيات

فصل المسؤوليات يمنع التداخل ويحمي بيانات التشغيل.

الدور نطاق الاستخدام أهم الصلاحيات
مدير الإدارة الكاملة للنظام. إدارة المحطات، المستخدمين، المراجعات، العملاء، الشيفتات، الرواتب، الإعدادات، وسجل التدقيق.
مشرف متابعة العمليات اليومية ومراجعة التقارير. عرض لوحات الإشراف، مراجعة التقارير، متابعة المحطات والطلبات، وتصدير بيانات محددة.
فني تنفيذ الفحص الميداني وتسجيل الزيارات. فتح صفحة المسح، إدخال تقارير المحطات، تسجيل الحضور والشيفتات، والتقارير اليومية.
عميل متابعة الطلبات والمحطات المصرح بها. تسجيل الطلبات، عرض بيانات الحساب، ومتابعة المحطات المرتبطة بحسابه حسب الإتاحة.

حماية صفحات النظام

الصفحات المحمية تمر عبر جلسة موثقة، ثم يتم التحقق من الدور المطلوب قبل عرض البيانات أو تنفيذ أي تعديل.

حماية العمليات الحساسة

عمليات الإنشاء والتعديل والاعتماد لا تعتمد على بيانات الدور القادمة من المتصفح، بل تقرأ المستخدم الحالي من الجلسة.

حسابات قابلة للإيقاف

يوجد مسار واضح للتعامل مع الحسابات المعطلة، وإلغاء الجلسة عند تعطيل المستخدم.

صلاحيات العميل منفصلة

العميل لا يدخل لوحات الإدارة أو مسارات الفنيين؛ يتم توجيهه إلى بوابته المخصصة.

الصلاحيات مبنية على Better Auth وجلسات محمية ودوال requireRole داخل الخادم.

البنية الفنية

تقنيات حديثة ومناسبة للنشر على بيئات إنتاجية.

المشروع مبني بتقنيات ويب حديثة تركز على الأداء، الأمان، سهولة التطوير، وقابلية التوسع. البيانات الحساسة والمنطق التشغيلي موجودان على الخادم، بينما الواجهة تقدم تجربة عربية سريعة وواضحة للمستخدم النهائي.

Next.js Server Components

قراءة البيانات والتحقق من الجلسات تتم على الخادم لتقليل كشف البيانات غير الضرورية.

Server Actions و API Routes

كل عملية محمية تمر عبر طبقة خادم تتحقق من الدور وتكتب سجل تدقيق عند الحاجة.

Drizzle Repositories

الوصول لقاعدة البيانات متمركز في Repository Layer، ما يسهل الاختبار والصيانة.

قاعدة بيانات مرنة

التطوير المحلي يستخدم SQLite، والنشر الإنتاجي يمكنه استخدام libSQL/Turso حسب البنية المختارة.

نماذج تحقق قوية

Zod و React Hook Form يقللان الأخطاء ويضمنان إدخال بيانات متوقعة قبل تنفيذ العمليات.

جاهزية نشر

الإعدادات الحساسة عبر Environment Variables، مع أوامر migration و seed وفحوصات جودة.

ملاحظة نشر: لا يتم تضمين أسرار أو روابط إنتاجية داخل الكود. يتم ضبط مفاتيح الجلسات، رابط النظام، وقاعدة البيانات من متغيرات البيئة وقت النشر.
البنية الفنية موثقة في README و ARCHITECTURE وملفات schema/repositories.

الأمان والاعتمادية

نقاط حماية مهمة تجعل المشروع مناسبًا للتشغيل الحقيقي.

جلسات آمنة

تسجيل الدخول يعتمد على Better Auth مع جلسات Cookies، مدة صلاحية قابلة للضبط، وسر جلسة من متغيرات البيئة.

صلاحيات من الخادم

النظام لا يثق في role أو uid القادم من الواجهة؛ الدور يقرأ من الجلسة الحالية داخل الخادم.

سجل تدقيق

العمليات الحساسة مثل إنشاء محطة، تعديل مستخدم، مراجعة تقرير، أو تحديث إعدادات تكتب Audit Log.

معاملات قاعدة بيانات

العمليات المركبة، خصوصًا إرسال التقرير، تتم داخل Transaction لتجنب حفظ نصف العملية فقط.

حماية من الاستخدام الخاطئ

يوجد تحقق من المدخلات، حدود لطول النصوص، حالات مراجعة محددة، وفلاتر تاريخية للتصدير.

سياسات متصفح

Middleware يضيف Content Security Policy ويفصل المسارات العامة عن المحمية ووضع الصيانة.

حالة فحص حالية: تم تشغيل أمر npm run typecheck أثناء إعداد هذا التقرير ونجح بدون أخطاء TypeScript.
  • توجد اختبارات للتحقق من الحسابات المعطلة، إعدادات البيئة، الإحداثيات، تصدير التقارير، QR، وجدولة الشيفتات.
  • توجد أوامر جاهزة للفحص: Typecheck، Lint، Test، Build، و Check شامل.
  • مسارات التحميل والخطأ موجودة للصفحات التشغيلية الرئيسية، ما يحسن تجربة المستخدم عند بطء البيانات أو حدوث خطأ.

تجربة المستخدم والهوية

واجهة عملية تناسب فرق التشغيل بدل الطابع التسويقي.

واجهة إيكوبست مصممة كأداة تشغيل يومية: قراءة سريعة، كثافة معلومات مناسبة، ألوان حالات واضحة، وتدفق استخدام مباشر للفني والمشرف والمدير. التصميم يعتمد على العربية أولًا ويدعم RTL في الصفحات الأساسية.

لغة عربية مختصرة

النصوص تركز على الفعل المطلوب والحالة التشغيلية، بدون شرح زائد داخل الواجهة.

ألوان حالات مفهومة

الأخضر للنشط والمعتمد، الأصفر للانتظار، الأحمر للرفض أو الخطر، والرمادي للحالات غير النشطة.

تنقل حسب الدور

كل مستخدم يرى لوحة ووجهات تتناسب مع مسؤولياته، ما يقلل التشتيت ويمنع الوصول الخاطئ.

دعم الوضع الداكن

يوجد نظام Theme مناسب لبيئات التشغيل المختلفة، مع الحفاظ على التباين ووضوح النص.

خرائط وإحداثيات

تدعم المحطات الإحداثيات، وتوجد صفحة خريطة للمحطات ذات المواقع المسجلة.

تجربة QR مباشرة

المسار العام للمسح يسهل الوصول السريع إلى محطة محددة من غير خطوات إدارية طويلة.

الهوية البصرية تعتمد على Slate و Teal ونمط واجهات تشغيلية هادئة.
10

متطلبات التشغيل والنشر

ما يحتاجه العميل أو فريق النشر لتشغيل النظام في بيئة إنتاجية.

البند المطلوب ملاحظات
قاعدة البيانات SQLite محليًا أو libSQL/Turso إنتاجيًا يتم تشغيل migrations قبل فتح النظام للمستخدمين.
أسرار المصادقة BETTER_AUTH_SECRET و AUTH_ROLE_COOKIE_SECRET قيم عشوائية قوية لا تقل عن 32 حرفًا.
رابط النظام BETTER_AUTH_URL و NEXT_PUBLIC_BASE_URL في الإنتاج يجب استخدام HTTPS ودومين ثابت.
مدير أول SEED_MANAGER_EMAIL / PASSWORD / NAME يتم إنشاء حساب المدير مرة واحدة ثم إزالة بيانات seed من البيئة.
التخزين الخارجي Adapter للصور عند اعتماد رفع الصور إنتاجيًا يمكن ربط S3 أو Cloudinary أو تخزين داخلي حسب سياسة العميل.
الفحص قبل التسليم Typecheck, Lint, Tests, Build يوجد أمر check شامل لتشغيل سلسلة الفحوصات.
نقطة تشغيلية: المشروع لا يحتاج إلى أسرار ثابتة داخل الكود. كل بيئة يمكن ضبطها بشكل مستقل، وهذا يسهل النقل من التطوير إلى staging ثم الإنتاج.
المتغيرات موثقة في .env.example، ولا يتم تضمين أي أسرار فعلية في التقرير.

خطة التسليم المقترحة

مسار عملي لتحويل النسخة الحالية إلى تشغيل إنتاجي مطمئن.

جلسة مراجعة مع العميل تأكيد أسماء الأدوار، نماذج التقارير، الحالات المطلوبة، وصلاحيات كل مستخدم.
تجهيز بيئة Staging نشر نسخة اختبار بدومين مؤقت، قاعدة بيانات منفصلة، وحسابات تجريبية.
اختبار دورة عمل كاملة إنشاء محطة، طباعة QR، إرسال تقرير، مراجعته، تصديره، ومراجعة Audit Log.
ضبط البيانات الأولية إضافة المناطق والمحطات والفريق والحسابات المطلوبة، ثم مراجعة الصلاحيات.
إطلاق Production تشغيل HTTPS، قاعدة إنتاجية، نسخ احتياطية، ومراقبة أخطاء الأداء والتشغيل.

توسعات قريبة مقترحة

تخزين صور إنتاجي، تحسين اختيار المواقع على الخريطة، تنبيهات داخلية للتقارير المنتظرة، ولوحة مؤشرات أعمق.

معايير قبول مختصرة

لا توجد أخطاء TypeScript، نجاح إنشاء تقرير كامل، حماية المسارات، تصدير CSV، وظهور Audit Log للعمليات الحساسة.

الخطة قابلة للتعديل حسب أولويات العميل وجدول الإطلاق.

الخلاصة

رسالة نهائية مناسبة لإرسال التقرير إلى العميل.

إيكوبست ليس مجرد لوحة بيانات؛ هو نظام تشغيل ميداني متكامل لمحطات الطعوم. المشروع يغطي المحطات، QR، تقارير الفحص، مراجعة المشرفين، إدارة العملاء، الحضور، الشيفتات، الرواتب، التقارير اليومية، التصدير، وسجل التدقيق. هذا يجعل النظام مناسبًا لشركة تحتاج إلى وضوح تشغيلي ومحاسبة دقيقة لكل زيارة وكل تعديل.

الانطباع المهني المقترح للعميل: المشروع جاهز للعرض كمنتج تشغيلي حقيقي، ومع مرحلة ضبط بيانات ونشر منظمة يمكن تحويله إلى بيئة إنتاجية مستقرة تخدم الإدارة والفريق الميداني والعملاء.

واضح

تدفقات استخدام قصيرة وحالات مفهومة لكل دور.

آمن

صلاحيات، جلسات، تدقيق، ومعاملات قاعدة بيانات.

قابل للتوسع

بنية Next.js و Drizzle تسمح بإضافة وحدات جديدة بدون إعادة بناء كاملة.

إعداد التقرير: مراجعة فنية لمشروع EcoPest الحالي، بصياغة عربية RTL مناسبة للإرسال إلى العميل.
